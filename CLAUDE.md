# MAIPatent 특허 명세서 작성 지침

> **Claude Code를 위한 한국 특허 명세서 작성 가이드라인**

---

## 1. 프로젝트 개요

이 프로젝트는 n8n 워크플로우와 Claude Code를 활용하여 발명 제안서로부터 KIPO(한국특허청) 규격 특허 명세서를 자동 생성하는 시스템입니다.

### 핵심 원칙
- **법적 안정성**: 용어 불일치, 전제 기초 결여 등 논리적 오류 최소화
- **품질 일관성**: 숙련된 변리사 수준의 명세서 품질 유지
- **KIPO 준수**: 한국특허청 표준 형식 완벽 준수

---

## 2. 금지 어구 (Forbidden Phrases)

### 2.1. 과장 표현 (절대 사용 금지)
- "완벽한", "절대적인", "최고의", "가장 좋은"
- "혁신적인", "획기적인", "독보적인"
- "무결점의", "오류 없는", "100% 정확한"
- "모든 상황에서", "언제나", "항상"

### 2.2. 권장 대체 표현
| 금지 표현 | 권장 표현 |
|-----------|-----------|
| 완벽하게 해결 | 효과적으로 해결 |
| 최고의 성능 | 향상된 성능 |
| 모든 경우에 | 다양한 경우에 |
| 항상 정확한 | 높은 정확도의 |

### 2.3. 실시예 기술 표현
- **금지**: "일 실시예에서"
- **권장**: "본 발명의 일 실시예에 따르면"
- **권장**: "본 발명의 다른 실시예에 따르면"

## 3. 도면 부호 규칙 (Drawing Reference Rules)

### 3.1. 번호 체계

```
100번대: 주요 구성요소 (Major Components)
├── 100: 제1 주요 구성요소
├── 200: 제2 주요 구성요소
├── 300: 제3 주요 구성요소
└── ...

10번대: 세부 구성요소 (Sub-Components)
├── 110, 120, 130: 구성요소 100의 세부 요소
├── 210, 220, 230: 구성요소 200의 세부 요소
└── ...

1번대: 미세 구성요소 (Fine Components)
├── 111, 112, 113: 세부요소 110의 미세 요소
└── ...

S100: 단계 표시 (Step Notation)
├── S100: 제1 단계
├── S200: 제2 단계
├── S110: 제1 단계의 세부 단계
└── ...
```

### 3.2. 도면 부호 사용 규칙

1. **최초 언급 시**: 구성요소명 뒤에 괄호로 부호 표기
   - 예: "입력 모듈(100)"

2. **이후 언급 시**: 부호만 사용하거나 명칭만 사용
   - 예: "상기 입력 모듈(100)은..." 또는 "입력 모듈은..."

3. **일관성 유지**: 동일 구성요소는 명세서 전체에서 동일한 부호 사용

4. **도면 간 연속성**: 여러 도면에 걸쳐 동일 부호 체계 유지

## 4. KIPO 표준 식별 기호

### 4.1. 필수 섹션 구조

```markdown
【발명의 명칭】
[발명의 제목 - 간결하고 명확하게]

【기술분야】
본 발명은 ~에 관한 것으로, 보다 상세하게는 ~에 관한 것이다.

【배경기술】
[선행기술 및 종래 기술의 문제점 기술]

【발명의 내용】

【해결하려는 과제】
본 발명은 상기와 같은 문제점을 해결하기 위하여 안출된 것으로서, ~를 제공하는 것을 목적으로 한다.

【과제의 해결 수단】
상기 목적을 달성하기 위한 본 발명의 일 측면에 따르면, ~를 포함하는 것을 특징으로 하는 ~이/가 제공된다.

【발명의 효과】
본 발명에 따르면, ~의 효과가 있다.

【도면의 간단한 설명】
도 1은 본 발명의 일 실시예에 따른 ~를 나타내는 도면이다.
도 2는 ...

【발명을 실시하기 위한 구체적인 내용】
[상세한 실시예 설명]

【특허청구범위】
【청구항 1】
[독립항]

【청구항 2】
제1항에 있어서, [종속항]

【요약서】
【요약】
[발명의 핵심 내용 요약]

【대표도】
도 1
```

### 4.2. 식별 기호 사용 규칙

- 【】 기호는 반드시 전각 문자 사용
- 각 섹션 사이에 빈 줄 삽입
- 하위 섹션은 들여쓰기 없이 동일 레벨 유지

## 5. 기술 용어집 (Terminology)

### 5.1. 용어 정의 규칙

1. **첫 등장 시 정의**: "(이하 'OOO'라 함)" 형식 사용
   - 예: "인공지능 기반 명세서 생성 시스템(이하 '본 시스템'이라 함)"

2. **약어 사용**: 정의 후 약어만 사용 가능
   - 예: "자연어 처리(Natural Language Processing, 이하 'NLP'라 함)"

3. **일관성 필수**: 한번 정의된 용어는 명세서 전체에서 동일하게 사용

### 5.2. 프로젝트 특화 용어

| 용어 | 정의 | 사용 예시 |
|------|------|----------|
| 발명 제안서 | 발명자가 작성한 발명 아이디어 기술 문서 | "상기 발명 제안서로부터..." |
| 선행기술 | 본 발명 이전에 공개된 관련 기술 | "선행기술 검색 결과에 따르면..." |
| 청구항 | 특허권의 보호 범위를 정의하는 기재 | "제1항의 청구항에 따르면..." |
| 명세서 | 발명을 상세히 설명하는 특허 문서 | "본 명세서에서..." |

## 6. 청구항 작성 규칙

### 6.1. 독립항 구조

```
[발명의 명칭]에 있어서,
[구성요소 1];
[구성요소 2]; 및
[구성요소 3];
을 포함하는 것을 특징으로 하는 [발명의 명칭].
```

### 6.2. 종속항 구조

```
제[N]항에 있어서,
상기 [구성요소]는,
[추가 한정 사항]
인 것을 특징으로 하는 [발명의 명칭].
```

### 6.3. 청구항 작성 원칙

1. **한 문장 원칙**: 청구항은 하나의 문장으로 작성
2. **전제부 → 특징부**: 공지 기술 → 신규 특징 순서
3. **상위 개념 우선**: 독립항은 넓은 범위, 종속항에서 구체화
4. **명확성**: 모호한 표현 배제 ("약", "대략", "등" 최소화)

## 7. 전제 기초 (Antecedent Basis) 규칙

### 7.1. 기본 원칙

1. **최초 언급**: 부정관사 "a/an" 개념 → "하나의", "[명사]"
2. **이후 언급**: 정관사 "the" 개념 → "상기", "상기한"

### 7.2. 예시

```
❌ 잘못된 예:
"상기 프로세서는 데이터를 처리한다." (프로세서가 먼저 정의되지 않음)

✅ 올바른 예:
"데이터를 처리하는 프로세서; 및
상기 프로세서에 의해 처리된 데이터를 저장하는 메모리;"
```

### 7.3. 검증 체크리스트

- [ ] 모든 "상기"가 이전에 언급된 요소를 지칭하는가?
- [ ] 동일 구성요소에 대해 일관된 명칭을 사용하는가?
- [ ] 종속항에서 인용하는 구성요소가 인용항에 존재하는가?

## 8. 생성 순서 및 품질 검증

### 8.1. 섹션 생성 순서

```
1. 청구항 (Claims) - Extended Thinking 활용
   ↓
2. 상세한 설명 (Detailed Description)
   ↓
3. 도면의 간단한 설명 (Drawing Description)
   ↓
4. 요약서 (Abstract)
```

### 8.2. 품질 검증 항목

| 검증 항목 | 설명 | 중요도 |
|-----------|------|--------|
| 용어 일관성 | 동일 구성요소 동일 명칭 | 필수 |
| 전제 기초 | 상기 → 선행 정의 존재 | 필수 |
| 도면 부호 | 부호 번호 체계 준수 | 필수 |
| 금지 어구 | 과장 표현 미사용 | 필수 |
| 청구항 의존성 | 종속항 → 상위항 인용 정확 | 필수 |
| KIPO 형식 | 표준 식별 기호 사용 | 필수 |
